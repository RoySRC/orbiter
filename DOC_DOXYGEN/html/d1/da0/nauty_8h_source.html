279 /*****************************************************************************
280 * *
281 * 16-bit, 32-bit and 64-bit versions can be selected by defining WORDSIZE. *
282 * The largest graph that can be handled has MAXN vertices. *
283 * Both WORDSIZE and MAXN can be defined on the command line. *
284 * WORDSIZE must be 16, 32 or 64; MAXN must be <= NAUTY_INFINITY-2; *
285 * *
286 * With a very slight loss of efficiency (depending on platform), nauty *
287 * can be compiled to dynamically allocate arrays. Predefine MAXN=0 to *
288 * achieve this effect, which is default behaviour from version 2.0. *
289 * In that case, graphs of size up to NAUTY_INFINITY-2 can be handled *
290 * if the the memory is available. *
291 * *
292 * If only very small graphs need to be processed, use MAXN<=WORDSIZE *
293 * since this causes substantial code optimizations. *
294 * *
295 * Conventions and Assumptions: *
296 * *
297 * A 'setword' is the chunk of memory that is occupied by one part of *
298 * a set. This is assumed to be >= WORDSIZE bits in size. *
299 * *
300 * The rightmost (loworder) WORDSIZE bits of setwords are numbered *
301 * 0..WORDSIZE-1, left to right. It is necessary that the 2^WORDSIZE *
302 * setwords with the other bits zero are totally ordered under <,=,>. *
303 * This needs care on a 1's-complement machine. *
304 * *
305 * The int variables m and n have consistent meanings throughout. *
306 * Graphs have n vertices always, and sets have m setwords always. *
307 * *
308 * A 'set' consists of m contiguous setwords, whose bits are numbered *
309 * 0,1,2,... from left (high-order) to right (low-order), using only *
310 * the rightmost WORDSIZE bits of each setword. It is used to *
311 * represent a subset of {0,1,...,n-1} in the usual way - bit number x *
312 * is 1 iff x is in the subset. Bits numbered n or greater, and *
313 * unnumbered bits, are assumed permanently zero. *
314 * *
315 * A 'graph' consists of n contiguous sets. The i-th set represents *
316 * the vertices adjacent to vertex i, for i = 0,1,...,n-1. *
317 * *
318 * A 'permutation' is an array of n ints repesenting a permutation of *
319 * the set {0,1,...,n-1}. The value of the i-th entry is the number to *
320 * which i is mapped. *
321 * *
322 * If g is a graph and p is a permutation, then g^p is the graph in *
323 * which vertex i is adjacent to vertex j iff vertex p[i] is adjacent *
324 * to vertex p[j] in g. *
325 * *
326 * A partition nest is represented by a pair (lab,ptn), where lab and ptn *
327 * are int arrays. The "partition at level x" is the partition whose *
328 * cells are {lab[i],lab[i+1],...,lab[j]}, where [i,j] is a maximal *
329 * subinterval of [0,n-1] such that ptn[k] > x for i <= k < j and *
330 * ptn[j] <= x. The partition at level 0 is given to nauty by the user. *
331 * This is refined for the root of the tree, which has level 1. *
332 * *
333 *****************************************************************************/
